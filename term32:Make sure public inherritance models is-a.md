# Make sure public inheritance models "is a"
## 1：继承的意义
当你想要用一个指针/引用统一处理多种不同类型的对象，并且这些对象之间存在“是一种”的语义关系时，只有继承能提供语言级别的类型适配性和动态绑定。
继承不使子类和基类成为“同一个类型”，而是使子类可以安全地当作基类类型使用。
当使用 public 继承的时候，编译器会自动地将public继承视作为is a 的关系，无论程序员的设计意图是否合理，会从多方面进行检查。
## 2：使用 public 继承 建模 is a 的关系，这是语言层面为类型系统提供的机制，使得子类型可以通过向基类转化视为同一个类型，并且配合虚函数可以实现多态
## 3：需要程序员来自己检查，使用 public 继承的时候，是否建模出is a 的关系，基类提供的方法可能不适用于派生类，需要程序员自己来检查
